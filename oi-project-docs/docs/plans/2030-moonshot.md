# 2030 Moonshot

## Premise

By 2030, the main interface to personal computing may not be search, apps, or chat windows.

It may be a persistent, user-owned agent that understands your projects and controls tools through embodied devices.

## North star

Agent Stick becomes the open personal agent OS.

## What that means

### Personal compute substrate

The agent is the primary continuity layer across:

- devices;
- tools;
- apps;
- services;
- projects;
- locations;
- models;
- time.

### Owned memory

User memory is portable, inspectable, editable, and not trapped inside a vendor chat history.

### Embodiment ecosystem

Any device can be an agent body if it implements the protocol:

- tiny voice pucks;
- watches;
- phones;
- room displays;
- cars;
- earbuds;
- desktop overlays;
- terminals;
- robots;
- accessibility devices.

### Capability economy

Tools are not apps. They are capabilities with:

- schemas;
- permissions;
- risk metadata;
- audit logs;
- rollback semantics;
- sandboxing;
- reputation.

### Character layer

Agents have user-owned visual/social presentation across devices, with semantic state clarity.

### Local/cloud hybridity

Users can run:

- fully local;
- local plus relay;
- managed cloud;
- private VPC;
- organization deployment.

The protocol and memory remain portable.

## Strategic wedges

1. Technical users building home agent rigs.
2. Developers using voice-to-agent workflows for coding.
3. Home Assistant/local-first community.
4. Accessibility and low-attention computing.
5. Small teams wanting private chief-of-staff agents.
6. Device manufacturers needing an agent embodiment layer.

## "Bigger than Google" interpretation

Google organizes the world's information around search and ads.

Agent Stick-style computing organizes a user's personal action space around intent, memory, and capability.

The huge version is not beating Google at search. It makes search less central.

## Platform primitives

- Agent identity;
- device registry;
- capability registry;
- task ledger;
- permission broker;
- memory substrate;
- embodiment protocol;
- character/theme system;
- user-owned cloud;
- skill marketplace;
- audit and governance.

## Risks

- safety/security failures;
- creepy UX;
- unreliable agents;
- overbroad permissions;
- cloud incumbents copy the obvious parts;
- open ecosystem gets flooded with malicious skills;
- users do not want to manage infrastructure;
- tiny device novelty wears off.

## Counter-positioning

Against phone assistants:

- local-first;
- user-owned;
- hackable;
- multi-device;
- explicit attention;
- real tools.

Against chatbots:

- persistent task state;
- embodied interaction;
- permissions;
- device routing;
- durable memory.

Against automation tools:

- natural language intent;
- dynamic delegation;
- personal context;
- multi-modal surfaces.

## 2030 demo

User walks into workshop, taps a tiny device:

```text
"Pick up where we left off on the synth firmware timing bug."
```

Agent:

- knows the project;
- checks current repo state;
- sees tests failing;
- routes oscilloscope notes to desktop;
- summarizes on wearable;
- proposes patch;
- asks approval on desk device;
- writes wiki note;
- schedules follow-up;
- never leaks secrets;
- leaves a complete audit trail.

That is the product.
