# Phased Roadmap

## Phase 0: Working sketch

Time: now

- M5Stick records on long-hold.
- Server runs STT/TTS.
- Responses cached on device.
- Double-tap playback.
- Basic screen status.

Value: validates interaction model.

## Phase 1: Protocol and state

Time: days

- Define DATP 0.1.
- Add command acks.
- Add device state reports.
- Add firmware state machine.
- Add task/event log.
- Add "what are you doing?"

Value: reliability and debuggability.

## Phase 2: Useful local assistant

Time: 1-2 weeks

- Device mute/dim/brightness.
- Todo capture.
- Wiki inbox append.
- Local project notes lookup.
- Short response routing.
- Basic dashboard.

Value: daily utility without dangerous actions.

## Phase 3: Tool broker and permissions

Time: 2-4 weeks

- Implement tool broker.
- Risk classes.
- Confirmation flow.
- Audit log.
- Scoped repo read tools.
- Scoped shell sandbox.
- Stop/revoke commands.

Value: safe enough for real work.

## Phase 4: Coding assistant workflows

Time: 1-2 months

- Repo assessment subagent.
- Branch/patch proposals.
- Test execution.
- Diff review on desktop.
- Stick approval cards.
- Wiki project context.

Value: turns voice capture into actual engineering leverage.

## Phase 5: Multi-device embodiment

Time: 2-3 months

- Device registry.
- Desktop/web client.
- Multiple Stick devices.
- Foreground routing.
- Attention policy.
- Private/public device classes.

Value: one agent, many bodies.

## Phase 6: Character/status system

Time: 3-4 months

- Built-in character packs.
- Semantic status states.
- Device-specific rendering.
- Pack validation.
- Generated pack experiment.

Value: legible, lovable status interface.

## Phase 7: Ecosystem bridges

Time: 4-6 months

- MCP adapter.
- OpenClaw adapter.
- Hermes Agent delegation.
- Rhasspy/Hermes MQTT bridge.
- Home Assistant tools.
- Skill sandbox MVP.

Value: plugs into agent ecosystem without surrendering control.

## Phase 8: Away-from-home

Time: 6-9 months

- Tailscale documented mode.
- Phone relay prototype.
- Hosted relay alpha.
- Remote risk policy.
- Push notification routing.

Value: agent is useful outside the house.

## Phase 9: Product hardening

Time: 9-18 months

- Signed firmware updates.
- Better pairing.
- Device fleet management.
- Installer.
- Backup/restore.
- Model/provider configuration.
- Observability.
- Test harness.

Value: usable by people who are not the author.

## Phase 10: Platform

Time: 18-36 months

- Capability marketplace.
- Audited skill registry.
- User-owned cloud/VPC.
- Device SDK.
- Theme/character marketplace.
- Agent memory portability.
- Multi-user households.
- Team/org mode.

Value: a real ecosystem.

## 2030 direction

- Personal agent OS.
- Owned memory substrate.
- Pluggable embodied devices.
- Safe tool economy.
- Open protocol for agent-device interaction.
- New default interaction layer above apps.
