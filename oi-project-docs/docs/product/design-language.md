# Design Language

## Tone

Calm, small, useful, non-invasive.

Avoid:

- always-on surveillance vibes;
- sci-fi command centre clutter;
- fake personality overreach;
- too much text on tiny screens.

## Visual language

Tiny device:

- big status character;
- tiny label;
- clear mode colour/icon;
- minimal animation;
- no scrolling walls of text.

Dashboard:

- active tasks;
- pending confirmations;
- device fleet;
- recent actions;
- memory diffs.

## Character style

Character should communicate state first, personality second.

Required readability:

- listening obvious;
- muted obvious;
- offline obvious;
- needs approval obvious;
- error obvious.

## Sound design

Default:

- no auto-speaking;
- gentle start/stop record tones optional;
- short cached TTS;
- optional mute/safe mode tone.

## Naming possibilities

Working names:

- Agent Stick
- Agent Fabric
- One Mind Many Bodies
- Pocket Chief
- Familiar OS
- Embodied Agent Runtime

Keep the repo name boring until the shape stabilizes.
